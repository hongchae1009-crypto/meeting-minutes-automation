# AI 기반 회의록 자동화 시스템 제안서

**작성일**: 2026년 3월 31일
**대상**: Confluence + Jira 사용 팀
**목표**: 음성 녹음 → AI 전사 → 회의록 자동 작성 + Jira 연동 파이프라인 구축

---

## 1. 개요

### 문제 정의
현재 회의록 작성 프로세스는 아래와 같은 비효율이 존재합니다:
- 녹음 후 수동으로 내용을 정리해야 함 (회의 1시간 = 정리 30분~1시간 소요)
- 작성자에 따라 회의록 품질/형식 불일치
- 결정사항·액션 아이템 누락 위험
- Jira 이슈 등록을 별도로 수동 처리
- 회의록과 Jira 이슈 간 연결이 끊어져 추적 어려움

### 제안 솔루션
**AI 기반 회의록 자동화 웹 대시보드** — 전사 텍스트 파일 업로드 한 번으로 Confluence 회의록 작성 + Jira 이슈 생성까지 자동 처리

## 2. 현황 분석

### 현재 워크플로우
```
[회의 녹음]
    ↓ (수동)
[AI 전사 도구로 텍스트 변환]  ← 이미 사용 중
    ↓ (수동 - 비효율 구간)
[내용 정리 및 요약]
    ↓ (수동)
[Confluence에 회의록 작성]
    ↓ (수동)
[Jira에 액션 아이템 등록]
```

### 전사 텍스트 포맷 분석

현재 AI 전사 결과물 형식 (`음성 260331_112812_original.txt` 기준):

```
화자  1  (00:00)
[발화 내용]

화자  2  (00:01)
[발화 내용]
...
```

- 화자 번호 + 타임스탬프 + 발화 내용 구조
- 여러 화자(화자 1, 2, 3) 구분됨
- UTF-16 인코딩 (Windows 환경)

## 4. 웹 대시보드 주요 기능

### Step 1: 파일 업로드 및 회의 정보 입력

| 입력 항목 | 설명 |
|-----------|------|
| 전사 텍스트 파일 | .txt 파일 drag & drop 업로드 |
| 회의명 | 예: "2026-03-31 주간 R&D 회의" |
| 회의 날짜/시간 | 자동 파일명 파싱 또는 수동 입력 |
| 화자 이름 매핑 | 화자 1 → 홍길동, 화자 2 → 김철수 |
| 관련 Jira 프로젝트 | 드롭다운 선택 |
| Confluence 스페이스 | 드롭다운 선택 |

> **화자 이름 매핑이 중요한 이유**: "화자 2가 다음 주까지 처리하기로 함" → "김철수 담당"으로 정확하게 변환됨

### Step 2: AI 처리 미리보기

```
┌─────────────────────────────────────────────────────┐
│  회의록 미리보기                     [편집] [재생성]  │
├─────────────────────────────────────────────────────┤
│  📋 회의 요약                                        │
│  R&D 프로젝트 역할 분담 및 5~6월 일정 논의.           │
│  XAI/PIN 기술 검토, 실시간 측정 정확도 목표 설정.     │
│                                                     │
│  ✅ 결정사항                                         │
│  • 5월 중 1단계 개발 완료 목표                        │
│  • 실시간 측정 정확도 목표 90%로 설정                  │
│                                                     │
│  📌 액션 아이템                                      │
│  [확실] 홍길동 | 4월 3일까지 | 발표 자료 준비         │
│  [확실] 김철수 | 5월 20일까지 | 소프트웨어 배포 준비   │
│  [추정] 미정  | 기한 없음   | XAI/PIN 분리 검토       │
│                                                     │
│  ⚠️ 검토 필요 항목 (3건)                [펼치기]     │
│  • (17:03) 화자2: "책 꺼내고 싶어..." → 참고문서?    │
│  • (18:18) 화자3: "링크 올리지 않아요?" → 업무 관련? │
│  • (09:14) 화자2: "응 잘 됩니다." → 포함 여부?       │
└─────────────────────────────────────────────────────┘
```

**핵심 UI 요소:**
- **[확실]** 태그: Claude가 명확한 지시사항으로 판단한 항목
- **[추정]** 태그: 암묵적·불명확한 할 일 (사용자 확인 필요)
- **⚠️ 검토 필요 항목**: 잡담인지 업무 발화인지 모호한 항목 따로 표시
- 각 항목 클릭 시 원본 타임스탬프 발화 확인 가능
- 섹션 간 드래그로 카테고리 이동 가능

### Step 3: 게시 설정 및 최종 확인

```
게시 대상 선택:
☑ Confluence 회의록 페이지 생성
  └ 스페이스: [R&D 팀] / 상위 페이지: [2026 회의록]

☑ Jira 이슈 생성 (액션 아이템 2건)
  └ 프로젝트: [RD-2026] / 이슈 유형: [Task]

☑ 기존 Jira 이슈에 댓글 추가
  └ 이슈 번호 입력: RD-142, RD-155

[게시하기]
```

### Step 4: 게시 결과 확인


## 6. Jira 연동 방식 상세

### 6-1. 액션 아이템 → 신규 Jira 이슈 자동 생성

```json
POST /rest/api/3/issue
{
  "fields": {
    "project": { "key": "RD" },
    "summary": "발표 자료 준비",
    "issuetype": { "name": "Task" },
    "assignee": { "accountId": "..." },
    "duedate": "2026-04-03",
    "description": {
      "회의 출처": "2026-03-31 R&D 주간회의",
      "원본 발화 타임스탬프": "06:01",
      "Confluence 링크": "https://..."
    }
  }
}
```

### 6-2. 기존 Jira 이슈에 회의 내용 댓글 추가

```json
POST /rest/api/3/issue/RD-142/comment
{
  "body": {
    "🗓 2026-03-31 회의에서 논의됨\n\n[관련 내용 요약]\n\n📄 전체 회의록: [Confluence 링크]"
  }
}
```

### 6-3. Confluence ↔ Jira 양방향 링크

- Confluence 페이지 내 "관련 Jira 이슈" 매크로 자동 삽입
- Jira 이슈 내 "원격 링크"로 Confluence 페이지 연결
- 어느 쪽에서든 서로 바로 이동 가능

## 7. AI 처리 한계 및 보완 방법

실제 회의 텍스트 검증을 통해 확인된 한계와 해결책:

| 한계 | 발생 빈도 | 보완 방법 |
|------|----------|----------|
| 비선형 주제 분산 | 모든 회의 | Claude의 전체 컨텍스트 처리로 자동 통합 (검증 완료) |
| 잡담/소음 필터링 실패 | ~10% | "검토 필요" 섹션으로 사용자 최종 판단 |
| 암묵적 액션 아이템 누락 | ~30% | [추정] 태그로 별도 표시 + 수동 추가 기능 |
| 담당자 불명확 | ~20% | 대시보드에서 수동 지정 |
| 대명사("이거", "그거") | 빈번 | 원문 타임스탬프 확인으로 맥락 파악 |
| 화자 이름 미상 | 항상 | 업로드 전 화자→이름 매핑 입력 |

> **결론**: AI가 80~90% 자동 처리하고, 나머지 10~20%는 인간 검토 단계에서 보완하는 **반자동 방식**이 최적

## 8. 기술 스택

| 영역 | 기술 | 선택 이유 |
|------|------|----------|
| Frontend | React + TypeScript | 빠른 UI 개발, 드래그&드롭 라이브러리 풍부 |
| Backend | Python FastAPI | Claude API, Atlassian API 연동 용이 |
| AI | Claude API (claude-sonnet-4-6) | 긴 회의 텍스트 처리, 한국어 성능 우수 |
| 회의록 | Confluence REST API v2 | 페이지 생성, 원격 링크, 매크로 지원 |
| 이슈 관리 | Jira REST API v3 | 이슈 생성, 댓글, 링크 API 완비 |
| 인증 | Atlassian API Token | 간단한 설정, 팀 공유 가능 |

## 10. 필요 설정 사항 (사전 준비)

### Atlassian 설정
- [ ] Atlassian Cloud 도메인 (예: `yourcompany.atlassian.net`)
- [ ] API 토큰 발급: https://id.atlassian.com/manage-profile/security/api-tokens
- [ ] Confluence 대상 스페이스 키 확인
- [ ] Jira 대상 프로젝트 키 확인
- [ ] 팀원 Atlassian 계정 ID 목록 (담당자 자동 지정용)

### Claude API 설정
- [ ] Anthropic API 키 발급: https://console.anthropic.com
- [ ] 사용량 한도 설정 (비용 관리)
