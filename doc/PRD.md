## 1. 배경 & 문제 정의

업무 미팅을 끝낸 뒤 사후 정리에 드는 시간이 길다. 음성 전사 도구로 텍스트는 빠르게 얻을 수 있지만, 거기서 다시 회의록 형식으로 다듬고, 결정사항·액션 아이템을 추출해 Confluence·Jira에 옮기는 작업이 반복된다. 이 단계를 AI로 한 번에 처리하고, 사용자는 검토·게시만 하도록 만든다.

---

## 2. 목표

- **G1.** 전사 텍스트 1개 → 구조화된 회의록(요약/논의/결정/액션) 자동 생성
- **G2.** Confluence 페이지 + Jira 이슈로 한 번에 게시 (옵션)
- **G3.** Atlassian 없이도 결과물을 가져갈 수 있는 내보내기 제공
- **G4.** 검토 필요 항목(잡담/추정)을 분리해 신뢰성 확보

### 비목표
- 음성 → 텍스트 전사 자체는 다루지 않음 (외부 도구 결과를 입력으로 받음)
- 다국어 회의록 (한국어 전사 우선)
- 실시간 회의록 (사후 일괄 처리 방식)

## 4. 기능 요구사항

### 4.1 5단계 마법사

| Step | 화면 | 핵심 동작 |
|---|---|---|
| 1 | 파일 업로드 | `.txt` 전사 파일 업로드 → 화자 목록 추출 |
| 2 | 화자 매핑 | "화자 1" → "홍길동" 매핑, 회의일·참석자 입력 |
| 3 | AI 처리 | Gemini로 회의록 생성 → 미리보기/편집 + **내보내기** |
| 4 | 게시 설정 | Confluence Space·Jira Project 선택, 토큰 입력 |
| 5 | 완료 | Confluence 페이지·Jira 이슈 링크 표시 |

### 4.2 전사 파일 입력 형식

- 인코딩: UTF-8, UTF-16 (BE/LE), CP949 자동 감지
- 패턴: `화자 N (MM:SS)` 라인 + 다음 줄에 발화 내용
- "화자" 한글 문자열이 본문에 반드시 포함

### 4.3 AI 생성 결과 구조

- **제목** (자동 추론, 편집 가능)
- **요약** (한 단락)
- **주요 논의** (토픽별 묶음)
- **결정사항** (리스트)
- **액션 아이템** (담당자/업무/기한/신뢰도 — `확실`/`추정`)
- **검토 필요 항목** (시간/화자/발화/이유 — 잡담·불명확 발화)
- **다음 회의 일정**

### 4.4 내보내기 (v1.1 추가)

Step 3에서 게시 단계를 거치지 않고 결과물을 가져갈 수 있다.

- **📋 마크다운 복사**: 클립보드에 회의록 전체를 마크다운으로
- **⬇ Markdown (.md)**: 파일 다운로드
- **⬇ JSON (.json)**: 데이터 보존용 (재처리·재활용)

### 4.5 Atlassian 연동

- **Confluence**: Space 선택 → 회의록 HTML 페이지 생성 (선택적 부모 페이지)
- **Jira**: 프로젝트 선택, Issue Type 지정, 담당자 매핑 (displayName → accountId)
- **댓글 추가**: 기존 이슈 키 입력 시 신규 생성 대신 댓글로 액션 아이템 추가
- **인증**: Atlassian API 토큰 (사용자가 화면에서 직접 입력)

## 5. 비기능 요구사항

| 항목 | 기준 |
|---|---|
| **응답 시간** | 60분 전사 기준 AI 처리 30–60초 |
| **보안** | API 토큰은 서버 환경변수만, Atlassian 토큰은 사용자가 매번 입력 (저장 안 함) |
| **CORS** | `*` 허용 (프론트가 어디서든 호출 가능) |
| **무료 운영** | Vercel Hobby + Render Free + Gemini 무료 티어 |
| **인코딩** | UTF-16/UTF-8/CP949 자동 감지 |

## 7. 성공 지표

- 마법사 5단계 완주율 ≥ 70%
- 회의록 1건당 사용자 편집 횟수 평균 ≤ 5회 (= AI 결과의 1차 품질)
- 액션 아이템 "확실" 비율 ≥ 60%
- 내보내기 사용률 (게시 미사용 사용자 비율) — Atlassian 미보유 사용자 수용 지표
